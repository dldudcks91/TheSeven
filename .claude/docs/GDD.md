## 1. 게임 개요

서버 포트폴리오 목적의 멀티플레이어 전략 게임. 플레이어는 자신의 도시를 건설하고, 군사력을 키우며, 연맹을 통해 다른 플레이어와 협력/경쟁한다. 스토리 없는 순수 전략 콘텐츠.

### 핵심 게임플레이 루프

```
자원 수집
   ↓
건물 건설/업그레이드 → 생산 및 군사력 강화
   ↓
연구 수행 → 버프/유닛 티어 해금
   ↓
유닛 훈련 → 병력 강화
   ↓
연맹 참가 → 연맹 연구 기여 → 공동 강화
   ↓
영웅 육성 → 전투 공격력 보정
   ↓
맵 탐색 → NPC 사냥 (경험치/보상) / PvP 출진 → 자원 약탈
```

## 2. 자원 시스템 ✅

### 2.1 자원 종류

| 자원 | 코드 | 설명 | 주 용도 |
|------|------|------|---------|
| 식량 | food | 기본 자원 | 건물, 연구, 유닛 훈련 비용 |
| 목재 | wood | 기본 자원 | 건물, 연구 비용 |
| 석재 | stone | 기본 자원 | 건물, 연구 비용 |
| 골드 | gold | 기본 자원 | 모든 비용 |
| 루비 | ruby | 프리미엄 재화 | 미구현 (재화만 존재) |

### 2.2 자원 소비 규칙

- 자원은 원자적(Atomic) 연산으로 차감 — Race Condition 방지
- 부족 시 `success: false`로 반환, 서버 예외 없음
- 자원 취소/환불: 건물/연구 취소 시 소모된 자원 100% 반환

## 6. 연맹 시스템 ✅

### 6.1 개요

- 플레이어들이 연합하여 공동 연구 수행 및 협력 가능
- 연맹 단위로 레벨, 경험치, 연구가 관리됨

### 6.2 직책 체계

| 직책 코드 | 이름 | 권한 |
|---------|------|------|
| 1 | 맹주 | 모든 권한 (해산, 가입방식 변경, 공지, 추방, 직책 변경, 연구 시작) |
| 2 | 부맹주 | 공지 작성, 추방, 직책 변경, 연구 추천/시작 |
| 3 | 간부 | 공지 작성, 추방, 연구 추천/시작 |
| 4 | 일반 | 기부, 연구 기부만 가능 |

### 6.3 가입 방식

| 방식 | 설명 |
|------|------|
| free | 자유 가입 (즉시 가입) |
| approval | 승인제 (맹주/간부 승인 필요) |

### 6.4 연맹 레벨

- 멤버의 자원 기부로 연맹 경험치(EXP) 획득
- 경험치 누적 시 연맹 레벨 상승
- 레벨에 따라 최대 멤버 수 증가

### 6.5 연맹 연구

| 연구 인덱스 | 이름 | 최대 레벨 | 효과 (예정) |
|-----------|------|---------|-----------|
| 8001 | 공격력 강화 | 5 | 연맹 멤버 전체 공격력 강화 |
| 8002 | 방어력 강화 | 5 | 연맹 멤버 전체 방어력 강화 |
| 8003 | 자원 생산 강화 | 5 | 연맹 멤버 전체 자원 생산 강화 |

**연맹 연구 흐름**:
1. 멤버들이 자원을 기부하여 연구별 EXP 적립
2. 간부 이상이 특정 연구를 "추천"으로 지정 (UI 표시용)
3. 경험치 충족 시 간부 이상이 "연구 시작" 버튼으로 레벨업 실행 (자동 레벨업 없음)

### 6.6 기부 비율

| 자원 | 연맹 EXP 비율 | 연맹 코인 비율 |
|------|-------------|------------|
| food | 1:100 | 1:500 |
| wood | 1:100 | 1:500 |
| stone | 1:100 | 1:500 |
| gold | 1:50 | 1:250 |
| ruby | 1:10 | 1:50 |

## 9. 전투 / 영웅 / 맵 시스템

> 상세 설계: [COMBAT.md](COMBAT.md) 참조

### 9.1 전투 시스템 ✅

- **맵**: 100×100 자유 좌표, Redis-only `map:positions` 해시로 유저 위치 관리 (DB 미저장)
- **행군**: 최대 3개 동시 출진, TaskWorker가 도착/귀환 타이머 처리
- **전투 방식**: 도착 시 즉시 교전 (이동 중 전투 없음), 1라운드 = 1초, 최대 50라운드
- **라운드 계산**: Python 구현 (`calculate_round()`), C++ 교체 후 벤치마크 비교 예정 (v0.7)
- **수비**: 오프라인 상태에서도 주둔 병력으로 자동 방어
- **결과**: 공격자 승리 시 자원 20% 약탈, 유닛 손실, 전투 리포트 DB 저장
- **실시간**: BattleWorker 1초 틱 → WS 개인 알림 + 맵 브로드캐스트

### 9.2 영웅 시스템 ✅

- 영웅은 행군에 1명 동행, 전투 공격력 보정 제공 (`hero_lv × 5%`)
- 동일 영웅 중복 출전 불가 (marching / battling / returning 상태 체크)
- 전투 승리 시 EXP 획득 → 레벨업 (`lv_up_exp = hero_lv × 100`)
- NPC 사냥 보상에도 영웅 EXP 포함
- **영웅 스킬**: CSV 구조 설계 완료, 전투 적용 미구현 (⏳)

### 9.3 NPC 사냥 시스템 ✅

- 맵에 배치된 NPC 거점(고블린 부족 등) 클릭 → 출진 → 전투
- NPC 난이도: `npc_info.csv` (npc_idx, tier, 병종/수량, EXP 보상)
- NPC 사망 후 리스폰 큐(`completion_queue:npc_respawn`) 등록 → 자동 부활
- 전투 결과: 영웅 EXP + 자원 보상 (PvP 약탈 없음)

### 9.4 맵 & 행군 시스템 ✅

- 맵 초기 접속 시 랜덤 빈 좌표 자동 배정 (Redis-only, DB 미저장)
- `map_info` API: 내 위치 + 반경 내 유저 + 활성 행군 전체 목록 반환
- WebSocket 실시간 브로드캐스트:
  - `map_march_start` — 행군 생성 (전체)
  - `map_march_update` — 행군 상태 변경 (marching → battling → returning) (전체)
  - `map_march_complete` — 귀환 완료 (전체)

## 11. 알려진 기술 부채

| 항목 | 설명 | 우선순위 |
|------|------|---------|
| DB 컬럼 ALTER 미지원 | `create_all()`로 신규 테이블은 자동 생성되지만 기존 테이블의 컬럼 추가/변경은 수동 ALTER 필요 | 중간 |
| 방어자 유닛 DB 폴백 미구현 | `battle_start()` 시 Redis 캐시 없으면 def_units 빈 dict로 처리 | 중간 |
| hero_skill 미적용 | hero_skill.csv 로드는 되나 전투 계산에 미반영 | 낮음 |
| NPC 서버 재시작 시 초기화 | Redis map:npcs는 메모리이므로 서버 재시작 시 NPC 위치 재배치됨 | 낮음 |
