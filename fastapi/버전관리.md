## v0.1 — 프레임워크 기반 · 로그인 · 자원

### 아키텍처 확립

- **Redis-first Cache-Aside** 패턴 도입
  - 모든 읽기/쓰기는 Redis 경유
  - MySQL은 영속성 보장용 (SyncWorker가 주기적 동기화)
- **APIManager 패턴** 확립
  - `POST /api { user_no, api_code, data }` 단일 엔드포인트
  - `api_map = { code: (ManagerClass, "method") }` 라우팅 테이블
  - Game Manager 메서드는 파라미터 없이 `self.user_no`, `self.data` 사용
- **BackgroundWorker** 기반 구조
  - BaseWorker → SyncWorker (카테고리별), TaskWorker (완료 이벤트)
- **WebSocketManager**: 작업 완료 시 실시간 알림

### DB 스키마 추가

```sql
-- StatNation 컬럼 추가
ALTER TABLE stat_nation ADD COLUMN map_x INT, ADD COLUMN map_y INT;

-- March 테이블 신규 (target_type, npc_id 포함)
CREATE TABLE march (
    march_id VARCHAR(36) PRIMARY KEY,  -- UUID
    user_no INT, target_user_no INT NULL, npc_id VARCHAR(36) NULL,
    target_type VARCHAR(10),           -- "user" | "npc"
    from_x INT, from_y INT, to_x INT, to_y INT,
    units TEXT,  -- JSON
    hero_idx INT NULL,
    march_speed FLOAT,
    departure_time DATETIME, arrival_time DATETIME, return_time DATETIME NULL,
    status VARCHAR(20),  -- marching/battling/returning/completed/cancelled
    battle_id VARCHAR(36) NULL
);

-- Battle 테이블 신규
CREATE TABLE battle (
    battle_id VARCHAR(36) PRIMARY KEY,  -- UUID
    march_id VARCHAR(36),
    attacker_user_no INT, defender_user_no INT NULL, npc_id VARCHAR(36) NULL,
    attacker_win BOOLEAN NULL,
    rounds INT NULL,
    attacker_losses TEXT NULL,  -- JSON
    defender_losses TEXT NULL,  -- JSON
    loot TEXT NULL,             -- JSON
    hero_exp INT NULL,
    created_at DATETIME
);

-- Hero 테이블 신규
CREATE TABLE hero (
    user_no INT,
    hero_idx INT,
    hero_lv INT DEFAULT 1,
    exp INT DEFAULT 0,
    PRIMARY KEY (user_no, hero_idx)
);
```
